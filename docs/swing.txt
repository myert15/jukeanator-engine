For Java 21, refine SongLibraryServiceImpl.getMusicBySearch() to be more like a traditional "fuzzy search" in that calculateSearchResultWeight() assigns weights as follows:
- Matched Leading Letter: +10 
- Consecutive Match: +5
- Unmatched Letter: -1
- Complete Match: +20
  
These matching criteria work together as follows:
- Matched Leading Letter: Algorithms heavily prioritize matches that start with the same letter as the search query. This increases the confidence that the results share the same context.
- Consecutive Match: Sequences of letters typed in the exact correct order (e.g., contiguous blocks) receive significant scoring bonuses over scattered or fragmented characters.
- Unmatched Letter: These represent errors, typos, or omitted characters. Algorithms penalize unmatched letters by calculating an "edit distance" (like insertions, deletions, or substitutions) to determine how far the query strays from the target word.
- Complete Match: A perfect 100% alignment where the search query is character-for-character identical to the target string, resulting in an edit distance of 0.

Use the attached Levenshtein Distance search algorithm example for guidance

In SongScanner, as an album's songs are iterated, a call is made to set the track number (if one can be extracted from the song filename)
If one cannot be extracted, then one is set by using the index of the song as it was processed in the parent album's childSongs collection.
The problem with this approach is that there could be many songs in an album that have the same embedded track number (see the attached screenshot
for an example).  The solution is to add a post-processing check to an album that does the following:

Item #1, If any duplicate track numbers, or non-consectuive track numbers are detected, then re-assign all track numbers to be based off a 
child song's order, if these child songs were added to a list (i.e. index of song in a list)

Item 2#, At the same time, call the getArtistName() as the songs are done in this post-processing step by an album, and if there are at least
two songs with a different artist value (embedded from the song filename), then set the transient Boolean property called "isCompilation"

The end result is that a track listing of an album's songs will have consecutive, non-duplicating track numbers and isCompilation will be 
true if at least two songs have different artists...
